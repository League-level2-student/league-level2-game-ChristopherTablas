Zombies!!!

by: Christopher Tablas

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game similar to the popular game minecraft. Minecraft is a game I used to play a lot and enjoy, which is why I made the player look like Steve and shoot arrows to zombies like you would in Minecraft. I wanted to make a game similar to other games I enjoyed.

2. What I Learned

From user testing I learned that I still had things to do even if I thought I was done. The feedback I received was to edit the hit boxes because they were off.

3. Architecture of My Game

My game had a total of 7 classes, each of them was needed to make the game playable. The arrow class was used to make the arrow, the game class was used to create the game, the GameObject class created the collision boxes, the GamePanel did a lot like create the menu screen and so much more, the ObjectManager class added the zombies and got the score, the player class created the player and made it move, the zombie class created the zombie. Together they made the game.

4. Technical Implementation

A part of the game i'm proud of are the hitboxes the objects have. They had terrible hitboxes and weren't even around the objects but after changing it they were close to perfect. I was stuck on how to change the boxes and had to ask for help. I now know how to set bounds to the collision box which is collisionBox.setBounds();

5. Most Challenging Part

The most challenging part of this project was coming up with a game to create. I had no idea what to do and it took me the whole week to figure out. I also struggled with making the player die/making the game end. It was because I used one = instead of two.

6. Future Fixes and Updates

An improvement I would make is the art because I think it could be better. I would like to create different mobs like some that shoot back and I would improve it by adding a timer so people know how much they survived.
